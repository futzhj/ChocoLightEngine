1 - 简介

Lua 是一种扩展编程语言，旨在支持通用的过程式编程和数据描述功能。 它还很好地支持面向对象编程、函数式编程和数据驱动编程。 Lua 旨在作为任何需要脚本语言的程序的强大、轻量级脚本语言。 Lua 以库的形式实现，使用 纯 C 语言编写（即 ANSI C 和 C++ 的通用子集）。

作为扩展语言，Lua 没有"主"程序的概念： 它只能嵌入到宿主客户端中， 称为嵌入程序或简称宿主。 宿主程序可以调用函数来执行一段 Lua 代码， 可以读写 Lua 变量， 并可以注册 C 函数供 Lua 代码调用。 通过使用 C 函数，Lua 可以扩展以适应各种不同的领域， 从而创建共享语法框架的定制编程语言。 Lua 发布包中包含一个名为 lua 的示例宿主程序， 它使用 Lua 库提供了一个完整的独立 Lua 解释器。

Lua 是自由软件， 并且像往常一样不提供任何保证， 具体内容见其许可证。 本手册描述的实现可在 Lua 官方网站 www.lua.org 获取。

和其他参考手册一样， 本文件在某些地方可能较为枯燥。 关于 Lua 设计背后的决策， 请参阅 Lua 网站上的技术论文。 如需详细了解 Lua 编程， 请参阅 Roberto 的著作 《Programming in Lua（第二版）》。

2 - 语言

本节描述 Lua 的词法、语法和语义。 换句话说， 本节描述了哪些记号是有效的， 它们如何组合， 以及这些组合的含义。

语言结构将使用常见的扩展 BNF 表示法进行说明， 其中 {a} 表示 0 个或多个 a， [a] 表示可选的 a。 非终结符用 non-terminal 表示， 关键字用 kword 表示， 其他终结符用 `=´` 表示。 Lua 的完整语法见本手册末尾的 第8节。

2.1 - 词法约定

名称（也称为标识符） 在 Lua 中可以是任意由字母、数字和下划线组成的字符串， 但不能以数字开头。 这与大多数语言中名称的定义一致。 （字母的定义取决于当前的区域设置： 任何被当前区域设置视为字母的字符都可以用作标识符。） 标识符用于为变量和表字段命名。

以下关键字是保留字， 不能用作名称： 

     and       break     do        else      elseif
     end       false     for       function  if
     in        local     nil       not       or
     repeat    return    then      true      until     while

Lua 区分大小写： and 是保留字，但 And 和 AND 是两个不同且有效的名称。 按照惯例，以下划线加大写字母开头的名称（如 _VERSION） 保留给 Lua 内部使用的全局变量。

以下字符串表示其他记号： 

     +     -     *     /     %     ^     #
     ==    ~=    <=    >=    <     >     =
     (     )     {     }     [     ]
     ;     :     ,     .     ..    ...

字符串字面量可以用成对的单引号或双引号括起来， 并可包含如下 C 风格的转义序列： '\a'（响铃）， '\b'（退格）， '\f'（换页）， '\n'（换行）， '\r'（回车）， '\t'（水平制表符）， '\v'（垂直制表符）， '\\'（反斜杠）， '\"'（双引号）， '\''（单引号）。 此外，反斜杠后跟真实换行符会在字符串中插入换行。 字符串中的字符也可以用转义序列 \ddd 指定其数值， 其中 ddd 是最多三位的十进制数字。 （注意，如果数值转义后面紧跟数字，必须用三位数字表示。） Lua 字符串可以包含任意 8 位值，包括嵌入的零， 可用 '\0' 指定。

字符串字面量还可以用长格式定义， 即用长括号括起来。 我们定义n 级长左括号为一个左中括号，后跟 n 个等号，再跟一个左中括号。 因此，0 级长左括号写作 [[， 1 级写作 [=[，以此类推。 长右括号定义类似； 例如，4 级长右括号写作 ]====]。 长字符串以任意级别的长左括号开始， 以第一个同级别的长右括号结束。 这种括号形式的字面量可以跨多行， 不解析任何转义序列， 并忽略其他级别的长括号。 它们可以包含除同级别长右括号外的任何内容。

为方便起见， 当长左括号后紧跟换行时， 该换行不会包含在字符串中。 例如，在使用 ASCII 的系统中 （其中 'a' 的编码为 97， 换行编码为 10，'1' 编码为 49）， 下面五个字符串字面量等价： 

     a = 'alo\n123"'
     a = "alo\n123\""
     a = '\97lo\10\04923"'
     a = [[alo
     123"]]
     a = [==[
     alo
     123"]==]

数值常量可以带有可选的小数部分和可选的指数部分。 Lua 也接受以 0x 开头的十六进制整数常量。 有效的数值常量示例： 

     3   3.0   3.1416   314.16e-2   0.31416E1   0xff   0x56

注释以双连字符（--）开头， 只要不在字符串内。 如果 -- 后不是长左括号， 则为短注释，直到行尾结束。 否则为长注释，直到对应的长右括号结束。 长注释常用于临时屏蔽代码。

2.2 - 值与类型

Lua 是一种动态类型语言。 这意味着变量没有类型，只有值有类型。 语言中没有类型定义。 所有值都自带类型。

Lua 中所有值都是一等值。 这意味着所有值都可以存储在变量中， 作为参数传递给其他函数，或作为结果返回。

Lua 有八种基本类型： nil、boolean、number、 string、function、userdata、 thread 和 table。 Nil 是值 nil 的类型， 其主要特性是与任何其他值都不同； 它通常表示没有有用的值。 Boolean 是值 false 和 true 的类型。 nil 和 false 都会使条件为假； 其他任何值都为真。 Number 表示实数（双精度浮点数）。 （也可以很容易地构建使用其他内部数值表示的 Lua 解释器， 如单精度浮点或长整型；参见 luaconf.h 文件。） String 表示字符数组。 Lua 是 8 位无损的： 字符串可以包含任意 8 位字符， 包括嵌入的零（'\0'）（见2.1节）。

Lua 可以调用（并操作）用 Lua 编写的函数和用 C 编写的函数 （见2.5.8节）。

userdata 类型用于允许任意 C 数据存储在 Lua 变量中。 该类型对应一块原始内存， 在 Lua 中没有预定义的操作， 除了赋值和身份测试。 但通过使用元表， 程序员可以为 userdata 值定义操作 （见2.8节）。 Userdata 值不能在 Lua 中创建或修改， 只能通过 C API。 这保证了宿主程序拥有的数据的完整性。

thread 类型表示独立的执行线程， 用于实现协程（见2.11节）。 不要将 Lua 线程与操作系统线程混淆。 Lua 在所有系统上都支持协程， 即使这些系统不支持线程。

table 类型实现了关联数组， 即不仅可以用数字索引， 还可以用任何值（除了 nil）索引的数组。 表可以是异构的； 即它们可以包含所有类型的值（除了 nil）。 表是 Lua 唯一的数据结构机制； 它们可以用来表示普通数组、符号表、集合、记录、图、树等。 要表示记录，Lua 使用字段名作为索引。 语言通过提供 a.name 作为 a["name"] 的语法糖来支持这种表示。 Lua 有多种方便的方式创建表（见2.5.7节）。

与索引一样， 表字段的值可以是任意类型（除了 nil）。 特别地， 由于函数是一等值， 表字段可以包含函数。 因此表也可以携带方法（见2.5.9节）。

表、函数、线程和（完整的）userdata 值是对象： 变量实际上并不包含这些值， 而只是对它们的引用。 赋值、参数传递和函数返回 总是操作这些值的引用； 这些操作不会产生任何形式的拷贝。

库函数 type 返回描述给定值类型的字符串。

2.2.1 - 类型转换

Lua 在运行时提供字符串和数字值之间的自动转换。对字符串进行的任何算术操作都会尝试将该字符串转换为数字，遵循常规转换规则。反之，在需要字符串的地方使用数字时，数字会以合理的格式转换为字符串。如需完全控制数字到字符串的转换方式， 请使用字符串库中的 format 函数（见string.format）。

2.3 - 变量

变量是存储值的位置。 Lua 中有三种变量： 全局变量、局部变量和表字段。

单个名称可以表示全局变量或局部变量 （或函数的形式参数， 它是一种特殊的局部变量）： 

	var ::= Name

Name 表示标识符，定义见2.1节。

任何变量都被假定为全局变量，除非显式声明为局部变量（见2.4.7节）。 局部变量是词法作用域的： 在其作用域内定义的函数可以自由访问这些局部变量（见2.6节）。

在变量第一次赋值前，其值为 nil。

方括号用于索引表： 

	var ::= prefixexp `[´ exp `]´

对全局变量和表字段的访问可以通过元表改变其含义。 对索引变量 t[i] 的访问等价于调用 gettable_event(t,i)。 （gettable_event 函数的完整描述见2.8节。 该函数在 Lua 中未定义且不可调用， 这里只用于解释说明。）

语法 var.Name 只是 var["Name"] 的语法糖： 

	var ::= prefixexp `.´ Name

所有全局变量都作为普通 Lua 表的字段存在， 这些表称为环境表或简称环境（见2.9节）。 每个函数都有自己的环境引用， 因此该函数中的所有全局变量都指向这个环境表。 当创建函数时， 它会继承创建它的函数的环境。 要获取 Lua 函数的环境表， 请调用 getfenv。 要替换它， 请调用 setfenv。 （只能通过调试库操作 C 函数的环境；见5.9节。）

访问全局变量 x 等价于 _env.x， 而这又等价于 

     gettable_event(_env, "x")

其中 _env 是当前运行函数的环境。 （gettable_event 函数的完整描述见2.8节。 该函数在 Lua 中未定义且不可调用， 同样，_env 变量在 Lua 中也未定义。 这里只用于解释说明。）

2.4 - 语句

Lua 支持几乎传统的语句集合， 类似于 Pascal 或 C。 该集合包括赋值、控制结构、函数调用和变量声明。

2.4.1 - 代码块

Lua 的执行单元称为代码块（chunk）。 代码块就是一系列语句， 按顺序执行。 每个语句后面可以选择性地加分号： 

	chunk ::= {stat [`;´]}

没有空语句，因此 ';;' 是非法的。

Lua 将代码块视为一个匿名函数的函数体， 该函数参数数量可变（见2.5.9节）。 因此，代码块可以定义局部变量， 接收参数并返回值。

代码块可以存储在文件中，或作为字符串存储在宿主程序中。

执行代码块时，Lua 首先将代码块预编译为虚拟机指令，然后用虚拟机解释器执行编译后的代码。

代码块也可以预编译为二进制形式；详情见 luac 程序。源代码和编译后的程序可以互换；Lua 会自动检测文件类型并做相应处理。

2.4.2 - 代码段

代码段是语句的列表；在语法上，代码段和代码块是一样的： 

	block ::= chunk

代码段可以显式限定为单个语句： 

	stat ::= do block end

显式代码段有助于控制变量声明的作用域。有时也用于在其他代码段中间插入 return 或 break 语句（见2.4.4节）。

2.4.3 - 赋值

Lua 允许多重赋值。因此，赋值语法定义了左侧的变量列表和右侧的表达式列表，两者用逗号分隔： 

	stat ::= varlist `=´ explist
	varlist ::= var {`,´ var}
	explist ::= exp {`,´ exp}

表达式详见2.5节。

在赋值前，值列表会调整为变量列表的长度。如果值多于变量，多余的值会被丢弃；如果值少于变量，列表会用 nil 补足。如果表达式列表以函数调用结尾，则该调用返回的所有值都会进入值列表，然后再调整（除非调用被括号包裹；见2.5节）。

赋值语句会先计算所有表达式，然后再执行赋值。因此： 

     i = 3
     i, a[i] = i+1, 20

会将 a[3] 设为 20，而不会影响 a[4]，因为 a[i] 中的 i 在赋值前已被计算为 3。 同理， 

     x, y = y, x

会交换 x 和 y 的值， 

     x, y, z = y, z, x

会循环置换 x、y、z 的值。

对全局变量和表字段的赋值可以通过元表改变其含义。对索引变量 t[i] = val 的赋值等价于 settable_event(t,i,val)。（settable_event 的完整描述见2.8节。该函数在 Lua 中未定义且不可调用，仅用于解释说明。）

对全局变量 x = val 的赋值等价于 _env.x = val，又等价于 

     settable_event(_env, "x", val)

其中 _env 是当前运行函数的环境。（_env 变量在 Lua 中未定义，仅用于解释说明。）

2.4.4 - 控制结构

控制结构 if、while 和 repeat 具有常见含义和熟悉语法： 

	stat ::= while exp do block end
	stat ::= repeat block until exp
	stat ::= if exp then block {elseif exp then block} [else block] end

Lua 还有两种 for 语句（见2.4.5节）。

控制结构的条件表达式可以返回任意值。false 和 nil 被视为假，其他所有值都为真（包括数字 0 和空字符串）。

在 repeat–until 循环中，内部代码段不会在 until 关键字处结束，而是在条件判断后结束。因此，条件可以引用循环块内声明的局部变量。

return 语句用于从函数或代码块（本质上是函数）返回值。函数和代码块可以返回多个值，因此 return 语句的语法为： 

	stat ::= return [explist]

break 语句用于终止 while、repeat 或 for 循环，跳转到循环后的下一个语句： 

	stat ::= break

break 只会结束最内层的循环。

return 和 break 语句只能作为代码段的最后一条语句。如果确实需要在代码段中间 return 或 break，可以用显式内部代码段包裹，如 do return end 和 do break end，这样 return 和 break 就是其（内部）代码段的最后一条语句。

2.4.5 - for 语句

for 语句有两种形式：数值型和泛型。

数值型 for 循环在控制变量按算术级数变化时重复执行代码块。语法如下： 

	stat ::= for Name `=´ exp `,´ exp [`,´ exp] do block end

block 会在 name 从第一个 exp 的值开始，按第三个 exp 的步长变化，直到超过第二个 exp。更精确地说： 

     for v = e1, e2, e3 do block end

等价于： 

     do
       local var, limit, step = tonumber(e1), tonumber(e2), tonumber(e3)
       if not (var and limit and step) then error() end
       while (step > 0 and var <= limit) or (step <= 0 and var >= limit) do
         local v = var
         block
         var = var + step
       end
     end

注意：

  • 所有三个控制表达式只在循环开始前计算一次，且都必须为数字。
  • var、limit、step 是不可见变量，仅为说明用。
  • 如果第三个表达式（步长）省略，则默认为 1。
  • 可以用 break 跳出 for 循环。
  • 循环变量 v 只在循环体内有效，循环结束或跳出后无法再用。如需保留其值，请在跳出前赋值给其他变量。

泛型 for 语句用于遍历迭代器函数。每次迭代时，迭代器函数会被调用产生新值，直到新值为 nil 时停止。语法如下： 

	stat ::= for namelist in explist do block end
	namelist ::= Name {`,´ Name}

如： 

     for var_1, ···, var_n in explist do block end

等价于： 

     do
       local f, s, var = explist
       while true do
         local var_1, ···, var_n = f(s, var)
         var = var_1
         if var == nil then break end
         block
       end
     end

注意：

  • explist 只计算一次，结果为迭代器函数、状态和初始值。
  • f、s、var 是不可见变量，仅为说明用。
  • 可以用 break 跳出 for 循环。
  • 循环变量 var_i 只在循环体内有效，循环结束后无法再用。如需保留其值，请在跳出前赋值给其他变量。

2.4.6 - 函数调用作为语句

为了允许副作用，函数调用可以作为语句执行： 

	stat ::= functioncall

此时所有返回值都会被丢弃。函数调用详见2.5.8节。

2.4.7 - 局部声明

局部变量可以在代码段内任意位置声明。声明时可包含初始赋值： 

	stat ::= local namelist [`=´ explist]

如有初始赋值，其语义与多重赋值相同（见2.4.3节）。否则所有变量初始为 nil。

代码块也是代码段（见2.4.1节），因此局部变量可以在代码块外声明，其作用域延续到代码块末尾。

局部变量的可见性规则见2.6节。

2.5 - 表达式

Lua 的基本表达式如下： 

	exp ::= prefixexp
	exp ::= nil | false | true
	exp ::= Number
	exp ::= String
	exp ::= function
	exp ::= tableconstructor
	exp ::= `...´
	exp ::= exp binop exp
	exp ::= unop exp
	prefixexp ::= var | functioncall | `(´ exp `)´

数字和字符串见2.1节；变量见2.3节；函数定义见2.5.9节；函数调用见2.5.8节；表构造器见2.5.7节。可变参数表达式（...）只能在可变参数函数内直接使用，详见2.5.9节。

二元运算符包括算术运算符（见2.5.1节）、关系运算符（见2.5.2节）、逻辑运算符（见2.5.3节）和连接运算符（见2.5.4节）。一元运算符包括一元负号（见2.5.1节）、一元 not（见2.5.3节）和一元长度运算符（见2.5.5节）。

函数调用和可变参数表达式都可能产生多个值。如果表达式作为语句使用（仅限函数调用，见2.4.6节），其返回值列表会被调整为零元素，即所有返回值都被丢弃。如果表达式作为表达式列表的最后（或唯一）元素，则不会调整（除非调用被括号包裹）。在其他情况下，Lua 会将结果列表调整为一个元素，丢弃除第一个外的所有值。

示例： 

     f()                -- 调整为 0 个结果
     g(f(), x)          -- f() 调整为 1 个结果
     g(x, f())          -- g 得到 x 和 f() 的所有结果
     a,b,c = f(), x     -- f() 调整为 1 个结果（c 得到 nil）
     a,b = ...          -- a 得到第一个可变参数，b 得到第二个（如无则为 nil）
     a,b,c = x, f()     -- f() 调整为 2 个结果
     a,b,c = f()        -- f() 调整为 3 个结果
     return f()         -- 返回 f() 的所有结果
     return ...         -- 返回所有可变参数
     return x,y,f()     -- 返回 x、y 及 f() 的所有结果
     {f()}              -- 创建包含 f() 所有结果的表
     {...}              -- 创建包含所有可变参数的表
     {f(), nil}         -- f() 调整为 1 个结果

用括号括起来的表达式总是只产生一个值。因此，(f(x,y,z)) 总是单一值，即 f 返回的第一个值（如无则为 nil）。

2.5.1 - 算术运算符

Lua 支持常见的算术运算符：二元 +（加）、-（减）、*（乘）、/（除）、%（取模）、^（幂）；一元 -（取负）。如果操作数是数字或可转换为数字的字符串（见2.2.1节），则运算有通常含义。幂运算支持任意指数，例如 x^(-0.5) 计算 x 的平方根的倒数。取模定义为： 

     a % b == a - math.floor(a/b)*b

即为向下取整后的余数。

2.5.2 - 关系运算符

Lua 的关系运算符有： 

     ==    ~=    <=    >=    <     >

这些运算符的结果总是 false 或 true。

等于（==）会先比较操作数类型，类型不同则为 false，否则比较值。数字和字符串按常规比较，对象（表、userdata、线程、函数）按引用比较：只有同一个对象才相等。每次创建新对象（表、userdata、线程、函数）都与之前的对象不同。

你可以通过 "eq" 元方法改变 Lua 比较表和 userdata 的方式（见2.8节）。

2.2.1节 的转换规则不适用于等于比较。因此 "0"==0 为 false，t[0] 和 t["0"] 是表中的不同项。

~= 运算符正好是等于（==）的否定。

大小比较运算符的工作方式如下：如果两个参数都是数字，则按数字比较；否则如果都是字符串，则按当前区域设置比较；否则 Lua 会尝试调用 "lt" 或 "le" 元方法（见2.8节）。a > b 会转为 b < a，a >= b 转为 b <= a。

2.5.3 - 逻辑运算符

Lua 的逻辑运算符有 and、or 和 not。与控制结构类似（见2.4.4节），所有逻辑运算符都将 false 和 nil 视为假，其他都为真。

not 总是返回 false 或 true。and 返回第一个参数（如为 false 或 nil），否则返回第二个参数。or 返回第一个参数（如不为 nil 和 false），否则返回第二个参数。and 和 or 都是短路运算符，即只有必要时才计算第二个操作数。示例： 

     10 or 20            --> 10
     10 or error()       --> 10
     nil or "a"          --> "a"
     nil and 10          --> nil
     false and error()   --> false
     false and nil       --> false
     false or nil        --> nil
     10 and 20           --> 20

（本手册中 --> 表示前面表达式的结果。）

2.5.4 - 连接运算符

Lua 的字符串连接运算符是两个点（'..'）。如果两个操作数都是字符串或数字，则按2.2.1节的规则转换为字符串，否则会调用 "concat" 元方法（见2.8节）。

2.5.5 - 长度运算符

长度运算符用一元 # 表示。字符串的长度是其字节数（即每个字符一个字节时的常规含义）。

表 t 的长度定义为任意整数索引 n，使得 t[n] 不为 nil 且 t[n+1] 为 nil；如果 t[1] 为 nil，则 n 可为 0。对于从 1 到 n 都非 nil 的常规数组，长度就是 n，即最后一个值的索引。如果数组有"空洞"（即中间有 nil），则 #t 可以是任何紧接 nil 之前的索引（即可能将任何 nil 视为数组结尾）。

2.5.6 - 优先级

Lua 的运算符优先级如下表，从低到高： 

     or
     and
     <     >     <=    >=    ~=    ==
     ..
     +     -
     *     /     %
     not   #     - (一元)
     ^

如常规，可用括号改变表达式优先级。连接（'..'）和幂（'^'）为右结合，其他二元运算符为左结合。

2.5.7 - 表构造器

表构造器是创建表的表达式。每次计算构造器都会创建新表。构造器可用于创建空表或初始化部分字段。通用语法如下： 

	tableconstructor ::= `{´ [fieldlist] `}´
	fieldlist ::= field {fieldsep field} [fieldsep]
	field ::= `[´ exp `]´ `=´ exp | Name `=´ exp | exp
	fieldsep ::= `,´ | `;´

[exp1] = exp2 形式的字段会向新表添加键为 exp1、值为 exp2 的项。name = exp 等价于 ["name"] = exp。exp 形式等价于 [i] = exp，其中 i 是从 1 开始的连续整数。其他格式的字段不会影响计数。例如： 

     a = { [f(1)] = g; "x", "y"; x = 1, f(x), [30] = 23; 45 }

等价于： 

     do
       local t = {}
       t[f(1)] = g
       t[1] = "x"         -- 第1个 exp
       t[2] = "y"         -- 第2个 exp
       t.x = 1            -- t["x"] = 1
       t[3] = f(x)        -- 第3个 exp
       t[30] = 23
       t[4] = 45          -- 第4个 exp
       a = t
     end

如果列表最后一个字段是 exp，且为函数调用或可变参数表达式，则该表达式返回的所有值会依次进入列表（见2.5.8节）。如需避免此行为，请用括号包裹该表达式（见2.5节）。

字段列表可选结尾分隔符，便于机器生成代码。

2.5.8 - 函数调用

Lua 的函数调用语法如下： 

	functioncall ::= prefixexp args

函数调用时，先计算 prefixexp 和 args。如果 prefixexp 的值类型为 function，则调用该函数并传入参数。否则会调用 prefixexp 的 "call" 元方法，第一个参数为 prefixexp 的值，后跟原始参数（见2.8节）。

形式： 

	functioncall ::= prefixexp `:´ Name args

可用于调用"方法"。v:name(args) 等价于 v.name(v,args)，但 v 只会被计算一次。

参数语法如下： 

	args ::= `(´ [explist] `)´
	args ::= tableconstructor
	args ::= String

所有参数表达式在调用前都会被计算。f{fields} 等价于 f({fields})，即参数列表为单个新表。f'string'（或 f"string" 或 f[[string]]）等价于 f('string')，即参数列表为单个字符串。

Lua 的自由格式语法有个例外：不能在函数调用的 '(' 前换行。这样可避免语法歧义。例如： 

     a = f
     (g).x(a)

Lua 会将其视为单条语句 a = f(g).x(a)。如需两条语句，需加分号。如需调用 f，则不能在 (g) 前换行。

return functioncall 形式称为尾调用。Lua 实现了真正的尾调用（或尾递归）：尾调用时，被调用函数会复用调用者的栈空间，因此嵌套尾调用次数无限制。但尾调用会丢失调用者的调试信息。注意，只有 return 后面只有一个函数调用参数时才是尾调用；此时调用者返回的正好是被调用者的所有返回值。以下都不是尾调用： 

     return (f(x))        -- 结果调整为 1
     return 2 * f(x)
     return x, f(x)       -- 还有其他返回值
     f(x); return         -- 结果被丢弃
     return x or f(x)     -- 结果调整为 1

2.5.9 - 函数定义

函数定义语法如下： 

	function ::= function funcbody
	funcbody ::= `(´ [parlist] `)´ block end

以下语法糖简化了函数定义： 

	stat ::= function funcname funcbody
	stat ::= local function Name funcbody
	funcname ::= Name {`.´ Name} [`:´ Name]

如： 

     function f () body end

等价于： 

     f = function () body end


     function t.a.b.c.f () body end

等价于： 

     t.a.b.c.f = function () body end


     local function f () body end

等价于： 

     local f; f = function () body end

不等价于： 

     local f = function () body end

（只有当函数体内引用 f 时才有区别。）

函数定义是可执行表达式，其值类型为 function。Lua 预编译代码块时会预编译所有函数体。每次执行函数定义时，都会实例化（或闭包）该函数。该实例（或闭包）就是表达式的最终值。同一函数的不同实例可引用不同的外部局部变量，也可有不同的环境表。

参数作为局部变量，用实参初始化： 

	parlist ::= namelist [`,´ `...´] | `...´

函数调用时，实参列表会调整为形参列表长度，除非函数是可变参数函数（参数列表末尾有 ...）。可变参数函数不会调整参数列表，而是将多余参数通过可变参数表达式（同样写作 ...）传递。该表达式的值是所有多余实参的列表，类似多返回值函数。如果在其他表达式或表达式列表中间使用可变参数表达式，则其返回值列表会调整为一个元素；如在表达式列表末尾，则不调整（除非用括号包裹）。

示例： 

     function f(a, b) end
     function g(a, b, ...) end
     function r() return 1,2,3 end

参数与可变参数表达式的映射如下： 

     调用            形参
     f(3)             a=3, b=nil
     f(3, 4)          a=3, b=4
     f(3, 4, 5)       a=3, b=4
     f(r(), 10)       a=1, b=10
     f(r())           a=1, b=2
     g(3)             a=3, b=nil, ... -->  (无)
     g(3, 4)          a=3, b=4,   ... -->  (无)
     g(3, 4, 5, 8)    a=3, b=4,   ... -->  5  8
     g(5, r())        a=5, b=1,   ... -->  2  3

用 return 语句返回结果（见2.4.4节）。如函数体末尾未遇到 return，则无返回值。

冒号语法用于定义方法，即带有隐式额外参数 self 的函数。例如： 

     function t.a.b.c:f (params) body end

等价于： 

     t.a.b.c.f = function (self, params) body end

2.6 - 可见性规则

Lua 是词法作用域语言。变量的作用域从声明后第一条语句开始，持续到包含声明的最内层代码段结束。例如： 

     x = 10                -- 全局变量
     do                    -- 新代码段
       local x = x         -- 新的 'x'，值为 10
       print(x)            --> 10
       x = x+1
       do                  -- 另一个代码段
         local x = x+1     -- 又一个 'x'
         print(x)          --> 12
       end
       print(x)            --> 11
     end
     print(x)              --> 10  （全局变量）

注意，local x = x 这样的声明中新 x 尚未生效，因此右侧 x 指向外部变量。

由于词法作用域规则，内部函数可自由访问其作用域内定义的局部变量。内部函数引用的局部变量称为上值或外部局部变量。

每次执行 local 语句都会定义新的局部变量。例如： 

     a = {}
     local x = 20
     for i=1,10 do
       local y = 0
       a[i] = function () y=y+1; return x+y end
     end

该循环创建了十个闭包（即十个匿名函数实例）。每个闭包用不同的 y，但共享同一个 x。

2.7 - 错误处理

由于 Lua 是嵌入式扩展语言，所有 Lua 操作都从宿主程序的 C 代码调用 Lua 库函数开始（见lua_pcall）。Lua 遇到任何错误（如内存分配、类型、语法、运行时错误）时会抛出错误，即 long jump。受保护环境用 setjmp 设置恢复点，任何错误都会跳转到最近的恢复点。

大多数 API 函数可能抛出错误，如内存分配失败。每个函数文档会说明是否可能抛错。

在 C 函数内部可用 lua_error 抛出错误。

2.8 - 元表

Lua 中每个值都可以有元表。元表是普通 Lua 表，用于定义原值在某些特殊操作下的行为。通过设置元表中特定字段，可以改变值的操作行为。例如，非数字值参与加法时，Lua 会查找其元表中的 "__add" 字段，如有则调用该函数执行加法。

元表中的键称为事件，值称为元方法。如上例，事件为 "add"，元方法为执行加法的函数。

可用 getmetatable 查询任意值的元表。

可用 setmetatable 替换表的元表。其他类型的元表不能在 Lua 中更改（除非用调试库），需用 C API。

表和完整 userdata 有各自的元表（但多个表和 userdata 可共享元表）。其他类型的值每种类型只共享一个元表，如所有数字共用一个元表，所有字符串共用一个元表等。

元表控制对象在算术运算、比较、连接、长度、索引等操作下的行为，也可定义 userdata 被垃圾回收时调用的函数。Lua 为每种操作关联一个特定键（事件），当 Lua 对值执行操作时，会检查其元表是否有对应事件，如有则用该元方法控制操作。

元表控制的操作如下。每种操作用其对应名称标识，键为名称前加两个下划线，如 "__add"。这些操作的语义可用 Lua 函数描述解释器的执行方式。

此处 Lua 代码仅为说明，实际行为由解释器内置，效率更高。所有用到的函数（rawget、tonumber 等）见5.1节。如需获取对象的元方法，用表达式： 

     metatable(obj)[event]

即： 

     rawget(getmetatable(obj) or {}, event)

即访问元方法不会触发其他元方法，对无元表对象访问不会出错（结果为 nil）。

  • "add": + 运算。

    getbinhandler 函数定义了 Lua 如何选择二元操作的处理器。先尝试第一个操作数类型的处理器，如无则尝试第二个。 

         function getbinhandler (op1, op2, event)
       return metatable(op1)[event] or metatable(op2)[event]
     end

    用此函数，op1 + op2 的行为为： 

         function add_event (op1, op2)
       local o1, o2 = tonumber(op1), tonumber(op2)
       if o1 and o2 then  -- 两个操作数都是数字？
         return o1 + o2   -- 这里的'+'是原生加法
       else  -- 至少有一个不是数字
         local h = getbinhandler(op1, op2, "__add")
         if h then
           return (h(op1, op2))
         else
           error(···)
         end
       end
     end

  • "sub": - 运算。行为同 "add"。
  • "mul": * 运算。行为同 "add"。
  • "div": / 运算。行为同 "add"。
  • "mod": % 运算。行为同 "add"，原生操作为 o1 - floor(o1/o2)*o2。
  • "pow": ^（幂）运算。行为同 "add"，原生操作为 C 数学库的 pow。
  • "unm": 一元 - 运算。 
         function unm_event (op)
       local o = tonumber(op)
       if o then  -- 操作数是数字？
         return -o  -- 这里的'-'是原生取负
       else
         local h = metatable(op).__unm
         if h then
           return (h(op))
         else
           error(···)
         end
       end
     end

  • "concat": ..（连接）运算。 
         function concat_event (op1, op2)
       if (type(op1) == "string" or type(op1) == "number") and
          (type(op2) == "string" or type(op2) == "number") then
         return op1 .. op2  -- 原生字符串连接
       else
         local h = getbinhandler(op1, op2, "__concat")
         if h then
           return (h(op1, op2))
         else
           error(···)
         end
       end
     end

  • "len": # 运算。 
         function len_event (op)
       if type(op) == "string" then
         return strlen(op)         -- 原生字符串长度
       elseif type(op) == "table" then
         return #op                -- 原生表长度
       else
         local h = metatable(op).__len
         if h then
           return (h(op))
         else
           error(···)
         end
       end
     end

2.9 - 环境

除了元表外，thread、function 和 userdata 类型的对象还有另一个关联表，称为环境。环境和元表一样是普通表，多个对象可以共享同一个环境。

线程创建时会继承创建线程的环境。userdata 和 C 函数创建时会继承创建它们的 C 函数的环境。非嵌套 Lua 函数（由 loadfile、loadstring 或 load 创建）会继承创建线程的环境。嵌套 Lua 函数会继承创建它的 Lua 函数的环境。

userdata 关联的环境对 Lua 没有意义，仅便于程序员为 userdata 关联表。

线程关联的环境称为全局环境。它作为线程及其创建的非嵌套 Lua 函数的默认环境，并可被 C 代码直接访问（见3.3节）。

C 函数关联的环境可被 C 代码直接访问（见3.3节），也作为该函数创建的其他 C 函数和 userdata 的默认环境。

Lua 函数关联的环境用于解析函数内所有全局变量访问（见2.3节），也作为该函数创建的嵌套 Lua 函数的默认环境。

可用 setfenv 更改 Lua 函数或当前线程的环境，用 getfenv 获取。要操作其他对象（userdata、C 函数、其他线程）的环境，需用 C API。

2.10 - 垃圾回收

Lua 自动进行内存管理。你无需关心新对象的内存分配，也无需手动释放不再需要的对象。Lua 会定期运行垃圾回收器，回收所有不可达对象（即 Lua 无法访问的对象）。Lua 使用的所有内存都受自动管理：表、userdata、函数、线程、字符串等。

Lua 实现了增量标记-清除（mark-and-sweep）回收器。它用两个数值控制垃圾回收周期：回收暂停和回收步进倍率。两者单位为百分比（100 表示内部值为 1）。

回收暂停控制回收器在开始新周期前等待的时间。值越大，回收器越不激进。小于 100 表示不会等待。200 表示内存翻倍才开始新周期。

步进倍率控制回收器相对内存分配的速度。值越大，回收器越激进，但每步处理的数据也越多。小于 100 会导致回收器过慢，可能永远无法完成周期。默认 200，表示回收器速度是内存分配的"两倍"。

可用 C 的 lua_gc 或 Lua 的 collectgarbage 修改这些数值，也可直接控制回收器（如停止和重启）。

2.10.1 - 垃圾回收元方法

用 C API 可为 userdata 设置垃圾回收元方法（见2.8节），也称终结器。终结器允许你将 Lua 的垃圾回收与外部资源管理（如关闭文件、网络/数据库连接或释放自有内存）协调。

带有元表 __gc 字段的垃圾 userdata 不会被立即回收，而是被放入列表。回收后，Lua 会对列表中的每个 userdata 执行如下操作： 

     function gc_event (udata)
       local h = metatable(udata).__gc
       if h then
         h(udata)
       end
     end

每次垃圾回收周期结束时，会按创建顺序的逆序调用终结器。即最后创建的 userdata 的终结器最先被调用。userdata 本身只会在下一个回收周期被释放。

2.10.2 - 弱表

弱表是元素为弱引用的表。弱引用会被垃圾回收器忽略。换言之，如果对象只被弱引用引用，则会被回收。

弱表可以是弱键、弱值或两者皆弱。弱键表允许键被回收但值不会，键值都弱则两者都可回收。无论哪种，只要键或值被回收，整个键值对都会从表中移除。表的弱性由元表的 __mode 字段控制。__mode 包含字符 'k' 表示弱键，包含 'v' 表示弱值。

将表用作元表后，不应再修改其 __mode 字段，否则弱表行为未定义。

2.11 - 协程

Lua 支持协程（又称协作式多线程）。Lua 协程代表独立的执行线程。但与多线程系统不同，协程只有在显式调用 yield 函数时才会挂起。

用 coroutine.create 创建协程。参数为协程主函数。create 只创建协程并返回句柄（类型为 thread），不会启动协程。

首次用 coroutine.resume 调用协程（第一个参数为 coroutine.create 返回的线程），协程从主函数第一行开始执行。传给 coroutine.resume 的额外参数会传给主函数。协程启动后会一直运行，直到结束或挂起。

协程有两种终止方式：主函数正常返回（显式或隐式），或发生未捕获错误。第一种情况下，coroutine.resume 返回 true 及主函数返回值。出错时，coroutine.resume 返回 false 及错误信息。

协程通过调用 coroutine.yield 挂起。此时 coroutine.resume 会立即返回，即使 yield 发生在嵌套函数调用中（即不在主函数而在主函数直接或间接调用的函数中）。yield 时，coroutine.resume 也会返回 true 及传给 coroutine.yield 的值。下次 resume 时，协程会从 yield 处继续，coroutine.yield 返回 resume 传入的额外参数。

coroutine.wrap 也可创建协程，但返回的是 resume 协程的函数。调用该函数时，参数会传给 coroutine.resume。coroutine.wrap 返回 coroutine.resume 的所有值，除第一个布尔值外。与 coroutine.resume 不同，coroutine.wrap 不会捕获错误，错误会直接抛给调用者。

示例： 

     function foo (a)
       print("foo", a)
       return coroutine.yield(2*a)
     end
     
     co = coroutine.create(function (a,b)
           print("co-body", a, b)
           local r = foo(a+1)
           print("co-body", r)
           local r, s = coroutine.yield(a+b, a-b)
           print("co-body", r, s)
           return b, "end"
     end)
            
     print("main", coroutine.resume(co, 1, 10))
     print("main", coroutine.resume(co, "r"))
     print("main", coroutine.resume(co, "x", "y"))
     print("main", coroutine.resume(co, "x", "y"))

运行结果： 

     co-body 1       10
     foo     2
     
     main    true    4
     co-body r
     main    true    11      -9
     co-body x       y
     main    true    10      end
     main    false   cannot resume dead coroutine

3 - 应用程序接口（API）

本节描述 Lua 的 C API，即宿主程序与 Lua 通信可用的 C 函数集。所有 API 函数及相关类型和常量都在头文件 lua.h 中声明。

即使我们称之为"函数"，API 中的任何功能也可能以宏实现。这些宏对每个参数只使用一次（第一个参数总是 Lua 状态），不会产生隐藏副作用。

和大多数 C 库一样，Lua API 函数不会检查参数的有效性或一致性。但你可以通过在 luaconf.h 中为宏 luai_apicheck 提供合适定义来改变此行为。

3.1 - 栈

Lua 用虚拟栈在 C 与 Lua 之间传递值。栈中每个元素代表一个 Lua 值（nil、数字、字符串等）。

每当 Lua 调用 C 时，被调用函数会获得一个新栈，与之前的栈和仍在活动的 C 函数栈互不影响。该栈最初包含传给 C 函数的参数，C 函数将结果压入栈中以返回给调用者（见lua_CFunction）。

为方便起见，大多数 API 查询操作不严格遵循栈操作规则，而是可用索引引用栈中任意元素：正索引表示绝对栈位置（从 1 开始），负索引表示相对栈顶的偏移。具体地，若栈有 n 个元素，索引 1 表示第一个元素（最早压入），索引 n 表示最后一个元素，索引 -1 也表示栈顶元素，索引 -n 表示第一个元素。索引在 1 到栈顶之间（1 ≤ abs(index) ≤ top）时称为有效索引。

3.2 - 栈大小

与 Lua API 交互时，你需确保一致性。特别是，你需负责控制栈溢出。可用 lua_checkstack 增大栈空间。

每当 Lua 调用 C 时，会确保至少有 LUA_MINSTACK 个栈位可用。LUA_MINSTACK 定义为 20，因此通常无需担心栈空间，除非代码有循环不断压栈。

大多数查询函数可接受任意在可用栈空间内的索引，即通过 lua_checkstack 设置的最大栈空间内的索引。这样的索引称为可接受索引。更正式地： 

     (index < 0 && abs(index) <= top) ||
     (index > 0 && index <= stackspace)

注意 0 永远不是可接受索引。

3.3 - 伪索引

除非另有说明，任何接受有效索引的函数也可用伪索引，代表 C 代码可访问但不在栈上的 Lua 值。伪索引用于访问线程环境、函数环境、注册表和 C 函数的上值（见3.4节）。

线程环境（全局变量所在）总在伪索引 LUA_GLOBALSINDEX。当前 C 函数的环境总在伪索引 LUA_ENVIRONINDEX。

可用常规表操作访问和更改全局变量。例如，访问全局变量： 

     lua_getfield(L, LUA_GLOBALSINDEX, varname);

3.4 - C 闭包

创建 C 函数时，可为其关联一些值，形成C 闭包；这些值称为上值，函数每次调用都可访问（见lua_pushcclosure）。

每次调用 C 函数时，其上值位于特定伪索引。这些伪索引由宏 lua_upvalueindex 生成。第一个上值在 lua_upvalueindex(1)，依此类推。访问 lua_upvalueindex(n)（n 大于当前函数上值数但不超过 256）会产生可接受（但无效）索引。

3.5 - 注册表

Lua 提供注册表，一个预定义表，任何 C 代码都可用来存储需要的 Lua 值。该表总在伪索引 LUA_REGISTRYINDEX。任何 C 库都可往此表存数据，但应确保键与其他库不同以避免冲突。通常应用库名字符串或 C 对象地址的轻量 userdata 作为键。

注册表中的整数键由辅助库的引用机制使用，不应用于其他目的。

3.6 - C 中的错误处理

Lua 内部用 C 的 longjmp 处理错误（如用 C++ 也可用异常，见 luaconf.h）。Lua 遇到任何错误（如内存分配、类型、语法、运行时错误）时会抛出错误，即 long jump。受保护环境用 setjmp 设置恢复点，任何错误都会跳转到最近的恢复点。

大多数 API 函数可能抛出错误，如内存分配失败。每个函数文档会说明是否可能抛错。

在 C 函数内部可用 lua_error 抛出错误。

3.7 - 函数与类型

本节按字母顺序列出所有 C API 函数和类型。每个函数有如下指示：[-o, +p, x]

第一个字段 o 表示函数从栈弹出的元素数。第二个字段 p 表示函数压入栈的元素数（函数总是先弹出参数再压入结果）。x|y 表示函数可能压入（或弹出）x 或 y 个元素，取决于情况；问号 ? 表示无法仅凭参数确定弹/压元素数（如依赖栈内容）。第三个字段 x 表示函数是否可能抛错：- 表示绝不抛错，m 表示只因内存不足抛错，e 表示可能抛其他错误，v 表示可能主动抛错。

lua_Alloc

typedef void * (*lua_Alloc) (void *ud,
                             void *ptr,
                             size_t osize,
                             size_t nsize);

Lua 状态使用的内存分配函数类型。分配器功能类似 realloc，但不完全相同。参数为 ud（lua_newstate 传入的不透明指针）、ptr（被分配/重分配/释放的块指针）、osize（原始大小）、nsize（新大小）。ptr 仅当 osize 为 0 时为 NULL。nsize 为 0 时分配器必须返回 NULL；如 osize 不为 0，应释放 ptr 指向的块。nsize 不为 0 且 osize 为 0 时应如 malloc。nsize 和 osize 都不为 0 时如 realloc。Lua 假定 osize >= nsize 时分配器不会失败。

辅助库 luaL_newstate 用的分配器实现示例： 

     static void *l_alloc (void *ud, void *ptr, size_t osize,
                                                size_t nsize) {
       (void)ud;  (void)osize;  /* 未用 */
       if (nsize == 0) {
         free(ptr);
         return NULL;
       }
       else
         return realloc(ptr, nsize);
     }

假定 free(NULL) 无影响，realloc(NULL, size) 等价于 malloc(size)。ANSI C 保证这两点。

lua_atpanic

[-0, +0, -] 

lua_CFunction lua_atpanic (lua_State *L, lua_CFunction panicf);

设置新的 panic 函数并返回旧的。

如错误发生在任何受保护环境之外，Lua 会调用panic 函数，然后调用 exit(EXIT_FAILURE) 退出宿主应用。你的 panic 函数可通过不返回（如 long jump）避免退出。

panic 函数可访问栈顶的错误信息。

lua_call

[-(nargs + 1), +nresults, e] 

void lua_call (lua_State *L, int nargs, int nresults);

调用函数。

调用函数需遵循如下协议：先将被调用函数压栈，再按顺序压入参数（第一个参数先压）。最后调用 lua_call，nargs 为参数个数。调用时所有参数和函数值都会从栈弹出，函数结果会压入栈。结果数会调整为 nresults，除非 nresults 为 LUA_MULTRET，此时会压入所有结果。Lua 会确保返回值不会超出栈空间。结果按顺序压栈（第一个结果先压），因此调用后最后一个结果在栈顶。

被调用函数内部的任何错误都会向上传播（用 longjmp）。

以下示例演示宿主程序如何实现等价于 Lua 代码： 

     a = f("how", t.x, 14)

C 代码： 

     lua_getfield(L, LUA_GLOBALSINDEX, "f"); /* 被调用函数 */
     lua_pushstring(L, "how");                        /* 第1个参数 */
     lua_getfield(L, LUA_GLOBALSINDEX, "t");   /* 要索引的表 */
     lua_getfield(L, -1, "x");        /* 压入 t.x 结果（第2个参数） */
     lua_remove(L, -2);                  /* 移除栈上的 't' */
     lua_pushinteger(L, 14);                          /* 第3个参数 */
     lua_call(L, 3, 1);     /* 调用 'f'，3参数1结果 */
     lua_setfield(L, LUA_GLOBALSINDEX, "a");        /* 设置全局变量 'a' */

注意上述代码是"平衡"的：结束时栈恢复原状。这是良好编程习惯。

lua_CFunction

typedef int (*lua_CFunction) (lua_State *L);

C 函数类型。

为与 Lua 正确通信，C 函数需遵循如下协议：参数从 Lua 传入栈，按顺序（第一个参数先压）。函数开始时 lua_gettop(L) 返回参数个数，第一个参数（如有）在索引 1，最后一个参数在 lua_gettop(L)。返回值只需按顺序压栈（第一个结果先压），并返回结果数。栈上结果下方的其他值会被 Lua 丢弃。C 函数也可像 Lua 函数一样返回多个结果。

示例：以下函数接收任意数量的数值参数，返回平均值和总和： 

     static int foo (lua_State *L) {
       int n = lua_gettop(L);    /* 参数个数 */
       lua_Number sum = 0;
       int i;
       for (i = 1; i <= n; i++) {
         if (!lua_isnumber(L, i)) {
           lua_pushstring(L, "incorrect argument");
           lua_error(L);
         }
         sum += lua_tonumber(L, i);
       }
       lua_pushnumber(L, sum/n);        /* 第一个结果 */
       lua_pushnumber(L, sum);         /* 第二个结果 */
       return 2;                   /* 结果数 */
     }

     /* 表在索引 't' 处 */
     lua_pushnil(L);  /* 第一个键 */

       /* 使用 'key'（索引 -2）和 'value'（索引 -1） */
       printf("%s - %s\n",
              lua_typename(L, lua_type(L, -2)),
              lua_typename(L, lua_type(L, -1)));
       /* 移除 'value'，保留 'key' 以便下次迭代 */
       lua_pop(L, 1);
     }

lua_pcall

[-(nargs + 1), +(nresults|1), -]

以保护模式调用一个函数。

nargs 和 nresults 的含义与 lua_call 相同。如果调用过程中没有错误，lua_pcall 的行为与 lua_call 完全一致。然而，如果发生任何错误，lua_pcall 会捕获错误，将单个值（错误信息）压入栈中，并返回错误码。与 lua_call 一样，lua_pcall 总是移除函数及其参数。

通常，错误处理函数用于为错误信息添加更多调试信息，如堆栈回溯。由于在 lua_pcall 返回后栈已展开，因此无法再收集这些信息。

lua_pcall 成功时返回 0，否则返回以下错误码之一（在 lua.h 中定义）：

  • LUA_ERRRUN：运行时错误。
  • LUA_ERRMEM：内存分配错误。对于此类错误，Lua 不会调用错误处理函数。
  • LUA_ERRERR：运行错误处理函数时出错。

3.8 - 调试接口

Lua 没有内建调试功能，而是通过函数和钩子提供特殊接口。该接口允许构建各种调试器、分析器及其他需要解释器"内部信息"的工具。

  int nups;                   /* (u) number of upvalues */
  int linedefined;            /* (S) */
  int lastlinedefined;        /* (S) */
  char short_src[LUA_IDSIZE]; /* (S) */
  /* private part */
  other fields
} lua_Debug;

用于携带活动函数各种信息的结构体。lua_getstack 只填充该结构的私有部分，供后续使用。要用有用信息填充 lua_Debug 的其他字段，请调用 lua_getinfo。

lua_Debug 的字段含义如下：

  • source：若函数定义于字符串，则 source 为该字符串；若定义于文件，则 source 以 '@' 开头，后跟文件名。
  • short_src： source 的"可打印"版本，用于错误信息。
  • linedefined：函数定义起始行号。
  • lastlinedefined：函数定义结束行号。
  • what：若为 Lua 函数则为 "Lua"，C 函数为 "C"，主块为 "main"，尾调用函数为 "tail"。尾调用时 Lua 无其他信息。
  • currentline：函数当前执行的行号。无行信息时为 -1。
  • name：函数的合理名称。Lua 函数为一等值，不一定有固定名称。lua_getinfo 会根据调用方式查找合适名称，找不到则为 NULL。
  • namewhat：解释 name 字段。可能为 "global"、"local"、"method"、"field"、"upvalue" 或空字符串，取决于调用方式。
  • nups：函数的 upvalue 数量。

要获取函数调用信息，参数 ar 必须是由 lua_getstack 先前填充或作为钩子参数传递的有效激活记录（见 lua_Hook）。

要获取函数信息，需先将其压入栈顶，并让 what 字符串以 '>' 开头（此时 lua_getinfo 会弹出栈顶函数）。例如，想知道函数 f 定义在哪一行，可写： 

     lua_Debug ar;
     lua_getfield(L, LUA_GLOBALSINDEX, "f");  /* 获取全局 'f' */
     lua_getinfo(L, ">S", &ar);
     printf("%d\n", ar.linedefined);

  • 'u'：填充 nups 字段；

调试钩子函数类型。

每当调用钩子时，其 ar 参数的 event 字段会被设置为触发钩子的具体事件。Lua 用以下常量标识这些事件：LUA_HOOKCALL、LUA_HOOKRET、LUA_HOOKTAILRET、LUA_HOOKLINE 和 LUA_HOOKCOUNT。此外，对于行事件，currentline 字段也会被设置。要获取 ar 其他字段的值，钩子需调用 lua_getinfo。对于返回事件，event 可能为 LUA_HOOKRET（正常返回）或 LUA_HOOKTAILRET（尾调用返回）。尾调用时调用 lua_getinfo 无意义。

4 - 辅助库

辅助库为 C 与 Lua 交互提供了许多便捷函数。基础 API 提供了所有 C 与 Lua 交互的原语函数，而辅助库则为一些常见任务提供了更高级的函数。

辅助库的所有函数都定义在头文件 lauxlib.h 中，前缀为 luaL_。

辅助库中的所有函数都基于基础 API 构建，因此它们提供的功能都可以用基础 API 实现。

辅助库中有许多函数用于检查 C 函数参数，其名称总是 luaL_check* 或 luaL_opt*。这些函数在检查不通过时都会抛出错误。由于错误信息格式为参数（如"bad argument #1"），因此不应将这些函数用于其他栈值。

luaL_Buffer

typedef struct luaL_Buffer luaL_Buffer;

字符串缓冲区类型。

字符串缓冲区允许 C 代码分段构建 Lua 字符串。其用法模式如下：

  • 首先声明一个 luaL_Buffer 类型的变量 b。
  • 然后用 luaL_buffinit(L, &b) 初始化。
  • 接着用任意 luaL_add* 函数向缓冲区添加字符串片段。
  • 最后调用 luaL_pushresult(&b)。该调用会将最终字符串压入栈顶。

缓冲区正常操作期间会使用可变数量的栈槽。因此，使用缓冲区时不能假定栈顶位置。只要使用平衡，在缓冲区操作间可使用栈。即每次调用缓冲区操作时，栈应与上次缓冲区操作后处于同一层级（唯一例外是 luaL_addvalue）。调用 luaL_pushresult 后，栈恢复到初始化缓冲区时的层级，并将最终字符串置于栈顶。

引用是唯一的整数键。只要不手动向表 t 添加整数键，luaL_ref 保证返回的键唯一。可用 lua_rawgeti(L, t, r) 取回引用 r 指向的对象。luaL_unref 释放引用及其关联对象。

若栈顶对象为 nil，luaL_ref 返回常量 LUA_REFNIL。常量 LUA_NOREF 保证与 luaL_ref 返回的任何引用不同。

luaL_Reg

typedef struct luaL_Reg {
  const char *name;
  lua_CFunction func;
} luaL_Reg;

用于 luaL_register 注册函数的函数数组类型。name 为函数名，func 为函数指针。luaL_Reg 数组必须以 name 和 func 均为 NULL 的哨兵项结尾。

luaL_register

[-(0|1), +1, m] 

void luaL_register (lua_State *L,
                    const char *libname,
                    const luaL_Reg *l);

打开一个库。

若 libname 为 NULL，则将列表 l（见 luaL_Reg）中的所有函数注册到栈顶表中。

若 libname 非空，则 luaL_register 创建新表 t，将其设为全局变量 libname 的值、package.loaded[libname] 的值，并在其上注册列表 l 中的所有函数。若 package.loaded[libname] 或变量 libname 已有表，则重用该表。

无论哪种情况，函数都会将表留在栈顶。

luaL_typename

[-0, +0, -] 

const char *luaL_typename (lua_State *L, int index);

返回给定索引处值的类型名称。

luaL_typerror

[-0, +0, v] 

int luaL_typerror (lua_State *L, int narg, const char *tname);

生成如下错误信息： 

     location: bad argument narg to 'func' (tname expected, got rt)

其中 location 由 luaL_where 生成，func 为当前函数名，rt 为实际参数类型名。

luaL_unref

[-0, +0, -] 

void luaL_unref (lua_State *L, int t, int ref);

释放索引 t 处表中的引用 ref（见 luaL_ref）。该项会从表中移除，使被引用对象可被回收。引用 ref 也会被释放以便复用。

若 ref 为 LUA_NOREF 或 LUA_REFNIL，则什么也不做。

luaL_where

[-0, +1, m] 

void luaL_where (lua_State *L, int lvl);

将标识调用栈中第 lvl 层当前位置的字符串压入栈中。通常格式如下： 

     chunkname:currentline:

0 层为当前运行函数，1 层为调用当前函数的函数，依此类推。

此函数用于为错误信息构建前缀。

5 - 标准库

标准 Lua 库提供了许多通过 C API 实现的有用函数。其中一些为语言提供基本服务（如 type 和 getmetatable）；另一些提供对"外部"服务的访问（如 I/O）；还有一些本可用 Lua 实现，但因实用性或性能要求而用 C 实现（如 table.sort）。

所有库均通过官方 C API 实现，并作为独立 C 模块提供。目前 Lua 拥有如下标准库：

  • 基础库（含 coroutine 子库）；
  • 包管理库；
  • 字符串处理；
  • 表处理；
  • 数学函数（sin、log 等）；
  • 输入输出；
  • 操作系统功能；
  • 调试功能。

除基础库和包管理库外，每个库都将其所有函数作为全局表的字段或对象的方法提供。

要访问这些库，C 主程序应调用 luaL_openlibs，以打开所有标准库。也可分别调用 luaopen_base（基础库）、luaopen_package（包管理库）、luaopen_string（字符串库）、luaopen_table（表库）、luaopen_math（数学库）、luaopen_io（I/O 库）、luaopen_os（操作系统库）、luaopen_debug（调试库）。这些函数声明在 lualib.h，不应直接调用：应像调用其他 Lua C 函数一样调用它们，例如用 lua_call。

5.1 - 基础函数

基础库为 Lua 提供了一些核心函数。如果你的应用未包含该库，应仔细检查是否需要为其某些功能提供实现。

assert (v [, message])

当参数 v 的值为假（即 nil 或 false）时抛出错误；否则返回所有参数。message 为错误信息，缺省时为 "assertion failed!"

collectgarbage ([opt [, arg]])

此函数是垃圾收集器的通用接口。根据第一个参数 opt 执行不同操作：

  • "collect"：执行一次完整的垃圾回收循环（默认选项）。
  • "stop"：停止垃圾收集器。
  • "restart"：重启垃圾收集器。
  • "count"：返回 Lua 当前使用的总内存（KB）。
  • "step"：执行一次垃圾回收步进。步进"大小"由 arg 控制（值越大步进越多），具体方式未指定。若要精确控制步长需实验调整 arg。若步进完成一次回收循环，返回 true。
  • "setpause"：将 arg 设为收集器的新暂停值（见 §2.10），返回先前的暂停值。
  • "setstepmul"：将 arg 设为收集器的新步进乘数（见 §2.10），返回先前的步进乘数。

dofile ([filename])

打开指定文件并将其内容作为 Lua 块执行。无参数时，dofile 执行标准输入（stdin）内容。返回块返回的所有值。出错时，dofile 会将错误传播给调用者（即不在保护模式下运行）。

error (message [, level])

终止最后一个被保护调用的函数，并将 message 作为错误信息返回。error 永不返回。

通常 error 会在消息开头添加错误位置信息。level 参数指定如何获取错误位置。level 1（默认）为 error 被调用的位置，level 2 指向调用 error 的函数被调用的位置，依此类推。level 0 则不会在消息中添加位置信息。

_G

一个全局变量（非函数），保存全局环境（即 _G._G = _G）。Lua 本身不使用此变量；更改其值不会影响任何环境，反之亦然。（用 setfenv 更改环境。）

getfenv ([f])

返回函数当前使用的环境。f 可为 Lua 函数或指定栈层级的数字：1 层为调用 getfenv 的函数。若给定函数不是 Lua 函数，或 f 为 0，则返回全局环境。f 默认值为 1。

getmetatable (object)

若 object 无元表，返回 nil。否则，若元表有 "__metatable" 字段，则返回其值，否则返回该对象的元表。

ipairs (t)

返回三个值：迭代器函数、表 t 和 0，使得 

     for i,v in ipairs(t) do body end

会遍历 (1,t[1])、(2,t[2])、...，直到第一个缺失的整数键。

load (func [, chunkname])

用函数 func 获取代码片段并加载。每次调用 func 必须返回一个字符串，与前次结果拼接。返回空字符串、nil 或无值表示结束。

无错误时，返回编译后的函数；否则返回 nil 和错误信息。返回函数的环境为全局环境。

chunkname 用于错误信息和调试信息，缺省为 "=(load)"。

loadfile ([filename])

类似 load，但从文件 filename 或标准输入获取代码片段。

loadstring (string [, chunkname])

类似 load，但从给定字符串获取代码片段。

要加载并运行字符串，可用： 

     assert(loadstring(s))()

缺省时，chunkname 为给定字符串。

next (table [, index])

允许遍历表的所有字段。第一个参数为表，第二个参数为表中的索引。next 返回下一个索引及其值。第二参数为 nil 时，返回初始索引及其值。若为最后一个索引或空表，返回 nil。若省略第二参数，则视为 nil。可用 next(t) 检查表是否为空。

索引枚举顺序未指定，即使是数字索引。（如需按数字顺序遍历，用数值 for 或 ipairs。）

遍历期间若为表新增字段，next 行为未定义，但可修改已有字段。

可清空已有字段。

pairs (t)

返回三个值：next 函数、表 t 和 nil，使得 

     for k,v in pairs(t) do body end

会遍历表 t 的所有键值对。

遍历期间修改表的注意事项见 next。

pcall (f, arg1, ···)

以保护模式调用函数 f 及其参数。即 f 内部的任何错误不会被传播，而是被 pcall 捕获并返回状态码。第一个返回值为状态码（布尔值），成功为 true，且随后返回 f 的所有返回值。出错时返回 false 和错误信息。

print (···)

接收任意数量参数，将其值打印到 stdout，用 tostring 转为字符串。print 仅用于快速显示值，常用于调试。格式化输出请用 string.format。

rawequal (v1, v2)

判断 v1 是否等于 v2，不调用任何元方法。返回布尔值。

rawget (table, index)

获取 table[index] 的真实值，不调用任何元方法。table 必须为表，index 可为任意值。

rawset (table, index, value)

将 table[index] 的真实值设为 value，不调用任何元方法。table 必须为表，index 不能为 nil，value 可为任意 Lua 值。

此函数返回 table。

select (index, ···)

若 index 为数字，返回第 index 个参数之后的所有参数。否则 index 必须为字符串 "#"，此时 select 返回额外参数总数。

setfenv (f, table)

设置指定函数的环境。f 可为 Lua 函数或指定栈层级的数字，1 层为调用 setfenv 的函数。setfenv 返回该函数。

特殊情况：f 为 0 时，setfenv 更改当前线程的环境，此时无返回值。

setmetatable (table, metatable)

为指定表设置元表。（不能在 Lua 中更改其他类型的元表，只能用 C 更改。）metatable 为 nil 时移除元表。若原元表有 "__metatable" 字段，则抛出错误。

此函数返回 table。

tonumber (e [, base])

尝试将参数转换为数字。若参数已为数字或可转换为数字的字符串，则返回该数字，否则返回 nil。

可选参数指定进制，范围为 2~36。大于 10 时，字母 'A'（不区分大小写）代表 10，'B' 代表 11，依此类推，'Z' 代表 35。10 进制（默认）可有小数和可选指数部分（见 §2.1），其他进制仅接受无符号整数。

tostring (e)

接收任意类型参数并以合理格式转为字符串。完全控制数字转换请用 string.format。

若 e 的元表有 "__tostring" 字段，则调用其值并以 e 为参数，结果作为返回值。

type (v)

返回参数类型的字符串编码。可能结果有 "nil"（字符串，不是 nil 值）、"number"、"string"、"boolean"、"table"、"function"、"thread"、"userdata"。

unpack (list [, i [, j]])

返回给定表的元素。等价于： 
     return list[i], list[i+1], ···, list[j]

但上述代码只能用于固定元素数。默认 i 为 1，j 为列表长度（见 §2.5.5）。

_VERSION

一个全局变量（非函数），保存当前解释器版本字符串。当前内容为 "Lua 5.1"。

xpcall (f, err)

类似 pcall，但可自定义错误处理函数。

xpcall 以保护模式调用函数 f，用 err 作为错误处理器。f 内部的任何错误不会被传播，而是被 xpcall 捕获，并用原始错误对象调用 err，返回状态码。第一个返回值为状态码（布尔值），成功为 true，且随后返回 f 的所有返回值。出错时返回 false 和 err 的返回值。

5.2 - 协程操作

与协程相关的操作属于基础库的子库，包含在表 coroutine 中。协程的通用描述见 §2.11。

coroutine.create (f)

创建一个新协程，主体为 f。f 必须为 Lua 函数。返回新协程对象，类型为 "thread"。

coroutine.resume (co [, val1, ···])

启动或继续执行协程 co。首次 resume 时开始运行主体，val1 等作为主体函数参数。若协程已 yield，resume 会重启它，val1 等作为 yield 的返回值。

协程无错误运行时，resume 返回 true 及 yield 的所有值（若 yield）或主体函数返回的所有值（若结束）。出错时返回 false 和错误信息。

coroutine.running ()

返回当前运行的协程，主线程调用时返回 nil。

coroutine.status (co)

返回协程 co 的状态字符串："running"（正在运行，即调用 status）、"suspended"（在 yield 处挂起或未开始）、"normal"（活跃但未运行，即 resume 了其他协程）、"dead"（主体函数结束或出错）。

coroutine.wrap (f)

创建一个新协程，主体为 f。f 必须为 Lua 函数。返回一个函数，每次调用时 resume 协程。传递给该函数的参数作为 resume 的额外参数。返回 resume 的所有值（除第一个布尔值）。出错时抛出错误。

coroutine.yield (···)

挂起当前协程。协程不能在 C 函数、元方法或迭代器中运行。yield 的所有参数作为 resume 的额外返回值。

5.3 - 模块

package 库为 Lua 加载和构建模块提供基本功能。其两个函数 require 和 module 直接导出到全局环境，其他内容在表 package 中。

module (name [, ···])

创建模块。若 package.loaded[name] 有表，则该表为模块；否则若全局有同名表 t，则该表为模块；否则新建表 t，设为全局变量 name 和 package.loaded[name] 的值。还会初始化 t._NAME 为模块名，t._M 为模块自身，t._PACKAGE 为包名（模块名去掉最后一段）。最后，module 将 t 设为当前函数环境和 package.loaded[name] 的值，使 require 返回 t。

若 name 为复合名（用点分隔），module 会为每个部分创建（或重用）表。例如 a.b.c，则模块表存储在全局 a 的 b 字段的 c 字段。

该函数可在模块名后接可选参数，每个参数为应用于模块的函数。

require (modname)

加载指定模块。先查 package.loaded 表，若已加载则返回其值，否则查找加载器。

查找加载器时，require 依次调用 package.loaders 数组中的每个搜索器函数，参数为模块名。每个搜索器可返回加载器函数或说明未找到的字符串（或 nil）。Lua 默认有四个搜索器。

第一个搜索器查 package.preload 表。

第二个搜索器按 package.path 路径查找 Lua 库。路径为分号分隔的模板序列，每个模板中的问号会被模块名（点替换为目录分隔符）替换。例如路径为 "./?.lua;./?.lc;/usr/local/?/init.lua"，查找模块 foo 时会依次尝试 ./foo.lua、./foo.lc、/usr/local/foo/init.lua。

第三个搜索器按 package.cpath 路径查找 C 库。例如路径为 "./?.so;./?.dll;/usr/local/?/init.so"，查找模块 foo 时会依次尝试 ./foo.so、./foo.dll、/usr/local/foo/init.so。找到 C 库后，先用动态链接加载库，再查找名为 luaopen_ 加模块名（点替换为下划线）的 C 函数。若模块名有连字符，前缀会被去除。例如 a.v1-b.c，函数名为 luaopen_b_c。

第四个搜索器尝试一体化加载器，即在 C 路径中查找模块根名的库，找到后查找子模块的 open 函数。例如 a.b.c 会查找 a 的 C 库，并查找 luaopen_a_b_c。

找到加载器后，require 用模块名调用加载器。若加载器有返回值，则赋值给 package.loaded[modname]。若无返回值且未赋值，则 package.loaded[modname] 设为 true。最终返回 package.loaded[modname]。

加载或运行模块出错，或找不到加载器时，require 抛出错误。

package.cpath

require 查找 C 加载器时使用的路径。Lua 启动时用环境变量 LUA_CPATH 或 luaconf.h 中的默认路径初始化 C 路径。

package.loaded

require 用于记录已加载模块的表。require 某模块时，若 package.loaded[modname] 非 false，则直接返回其值。

package.loaders

require 用于控制如何加载模块的表。每个条目为搜索器函数。查找模块时，require 依次用模块名调用这些搜索器。搜索器可返回加载器函数或说明未找到的字符串（或 nil）。

第一个搜索器查 package.preload 表。

第二个搜索器按 package.path 路径查找 Lua 库。路径为分号分隔的模板序列，每个模板中的问号会被模块名（点替换为目录分隔符）替换。例如路径为 "./?.lua;./?.lc;/usr/local/?/init.lua"，查找模块 foo 时会依次尝试 ./foo.lua、./foo.lc、/usr/local/foo/init.lua。

第三个搜索器按 package.cpath 路径查找 C 库。例如路径为 "./?.so;./?.dll;/usr/local/?/init.so"，查找模块 foo 时会依次尝试 ./foo.so、./foo.dll、/usr/local/foo/init.so。找到 C 库后，先用动态链接加载库，再查找名为 luaopen_ 加模块名（点替换为下划线）的 C 函数。若模块名有连字符，前缀会被去除。例如 a.v1-b.c，函数名为 luaopen_b_c。

第四个搜索器尝试一体化加载器，即在 C 路径中查找模块根名的库，找到后查找子模块的 open 函数。例如 a.b.c 会查找 a 的 C 库，并查找 luaopen_a_b_c。这样一个包可将多个 C 子模块打包进一个库，每个子模块保留原有 open 函数。

package.loadlib (libname, funcname)

动态链接主程序与 C 库 libname。在库中查找函数 funcname 并返回为 C 函数（需遵循 lua_CFunction 协议）。

此为底层函数，完全绕过 package 和 module 系统。与 require 不同，不做路径查找也不自动加扩展名。libname 必须为 C 库完整文件名，funcname 必须为库导出的精确名称（可能依赖编译器和链接器）。

此函数不被 ANSI C 支持，仅部分平台可用（Windows、Linux、Mac OS X、Solaris、BSD 及支持 dlfcn 标准的其他 Unix）。

package.path

require 查找 Lua 加载器时使用的路径。Lua 启动时用环境变量 LUA_PATH 或 luaconf.h 中的默认路径初始化。环境变量值中的 ";;" 会被默认路径替换。

package.preload

用于存储特定模块加载器的表（见 require）。

package.seeall (module)

为 module 设置元表，其 __index 字段指向全局环境，使模块可继承全局环境的值。可作为 module 的选项。

5.4 - 字符串处理

本库提供字符串处理的通用函数，如查找、提取子串和模式匹配。Lua 中字符串索引从 1 开始（非 C 的 0）。索引可为负，表示从字符串末尾反向索引，-1 为最后一个字符。

字符串库所有函数都在表 string 中。还为字符串设置了元表，其 __index 字段指向 string 表，因此可用面向对象风格调用字符串函数，如 string.byte(s, i) 可写为 s:byte(i)。

字符串库假定单字节字符编码。

string.byte (s [, i [, j]])

返回 s[i]、s[i+1]、...、s[j] 的内部数值编码。i 默认 1，j 默认 i。

注意数值编码在不同平台间不一定可移植。

string.char (···)

接收零个或多个整数，返回长度等于参数数量的字符串，每个字符的内部编码等于对应参数。

注意数值编码在不同平台间不一定可移植。

string.dump (function)

返回给定函数的二进制表示字符串，后续用 loadstring 可还原为函数。function 必须为无 upvalue 的 Lua 函数。

string.find (s, pattern [, init [, plain]])

查找 pattern 在字符串 s 中首次出现的位置。找到则返回起止索引，否则返回 nil。可选参数 init 指定起始搜索位置，默认 1，可为负。第四参数 plain 为 true 时关闭模式匹配，仅做普通子串查找，此时 init 必须也给出。

string.format (formatstring, ...)

返回格式化字符串，语法与 C 的 printf 类似。formatstring 为格式模板，后续参数为要格式化的值。支持常见的格式控制符（如 %d、%s、%f 等）。

string.gmatch (s, pattern)

返回一个迭代器函数，每次调用返回字符串 s 中下一个匹配 pattern 的子串。

string.gsub (s, pattern, repl [, n])

返回字符串 s 的副本，将其中所有（或前 n 个）匹配 pattern 的子串替换为 repl。repl 可为字符串、表或函数。

string.len (s)

返回字符串 s 的长度（字符数）。

string.lower (s)

返回字符串 s 的小写形式。

string.match (s, pattern [, init])

在字符串 s 中查找 pattern 的第一个匹配项，返回捕获的子串。

string.rep (s, n)

返回字符串 s 重复 n 次的新字符串。

string.reverse (s)

返回字符串 s 的反转字符串。

string.sub (s, i [, j])

返回字符串 s 的子串，从第 i 个字符到第 j 个字符（包含 i 和 j）。

string.upper (s)

返回字符串 s 的大写形式。

5.5 - 表操作

本库提供了操作表（Lua 的唯一数据结构类型）的常用函数。所有函数都在 table 表中。

table.concat (table [, sep [, i [, j]]])

返回表 table 的 i 到 j 项（默认 1 到表长度），用分隔符 sep（默认空字符串）连接成一个字符串。只连接数值索引的元素。

table.insert (table [, pos,] value)

在表 table 的 pos 位置插入 value，后续元素顺序后移。若未指定 pos，则插入到表尾。

table.maxn (table)

返回表 table 的最大正整数索引（已废弃，建议用 #table 取长度）。

table.remove (table [, pos])

移除表 table 的 pos 位置元素，后续元素顺序前移。若未指定 pos，则移除表尾元素。返回被移除的元素。

table.sort (table [, comp])

对表 table 的元素进行原地排序。可选参数 comp 为自定义比较函数，若未指定则按小于号排序。

math.exp (x)

math.floor (x)

math.fmod (x, y)

math.huge

math.ldexp (m, e)

math.log (x [, base])

math.log10 (x)

math.max (x, ...)

math.min (x, ...)

5.7 - IO库

本库提供用于文件输入输出的函数。所有函数都在 io 表中。

io.close ([file])

关闭给定文件。若未指定 file，则关闭默认输出文件。成功返回 true，失败返回 nil 和错误信息。

io.flush ()

刷新默认输出文件的写入缓冲区。

io.input ([file])

若指定 file，将其设为默认输入文件。若未指定，则返回当前默认输入文件。

io.lines ([filename])

返回一个迭代器函数，每次调用返回文件的下一行。若未指定 filename，则从默认输入文件读取。

io.open (filename [, mode])

以指定模式打开文件，返回文件句柄。mode 默认为 "r"（只读），可选值有 "w"（写）、"a"（追加）、"r+"（读写）、"w+"（新建读写）、"a+"（追加读写）、以及 "b"（二进制模式）。打开失败返回 nil 和错误信息。

io.output ([file])

若指定 file，将其设为默认输出文件。若未指定，则返回当前默认输出文件。

io.popen (prog [, mode])

以只读或只写模式打开与进程 prog 的管道，返回文件句柄。

io.read (···)

从默认输入文件读取数据。参数可为数字（读取指定字节数）、"*n"（读取数字）、"*a"（读取全部内容）、"*l"（读取一行，默认）。

io.tmpfile ()

返回一个临时文件句柄（只读写，关闭后自动删除）。

io.type (obj)

判断 obj 是否为可用文件句柄。若是，返回 "file"；若已关闭，返回 "closed file"；否则返回 nil。

io.write (···)

将参数写入默认输出文件。

6 - 独立运行

Lua 可作为独立程序运行。标准 Lua 解释器可用命令行参数启动脚本、交互式输入或执行表达式。

6.1 命令行调用

Lua 解释器可通过命令行参数运行脚本文件、执行表达式或进入交互模式。常见参数有：

  • -e stat：执行字符串 stat 作为 Lua 代码。
  • -l name：加载模块 name。
  • -i：执行完脚本后进入交互模式。
  • -v：显示版本信息。
  • --：后续参数全部传递给脚本。

6.2 环境变量

Lua 启动时会读取一些环境变量，如 LUA_PATH、LUA_CPATH、LUA_INIT，用于配置模块搜索路径和初始化代码。

6.3 脚本参数

脚本可通过 arg 表访问命令行参数。arg[0] 为脚本名，arg[1] 开始为传递给脚本的参数。

7 - 兼容性

本章介绍 Lua 5.1 与早期版本之间的主要不兼容变更。

  • 模块系统和 require 机制有较大调整，建议参考新版文档。
  • 环境表（environment table）机制有变化，部分全局变量和函数的作用域不同。
  • 部分标准库函数已废弃或行为有变，如 table.maxn、setfenv、getfenv。
  • 元方法和元表的部分行为与旧版本不同。
  • 协程、错误处理等机制在边界条件下有细微差异。

更多详细兼容性说明，请参阅官方迁移指南。

7.1 - 语言变化

  • for 语句和 repeat 语句的隐式变量作用域有细微变化。
  • 长字符串/长注释语法（[[string]]）不再允许嵌套。若需嵌套可用新语法（[=[string]=]）。详见 luaconf.h 的 LUA_COMPAT_LSTR 选项。

7.2 - 库的变化

  • string.gfind 函数重命名为 string.gmatch。（详见 luaconf.h 的 LUA_COMPAT_GFIND 选项。）
  • 当 string.gsub 的第三个参数为函数时，若该函数返回 nil 或 false，则替换字符串为整个匹配内容，而非空字符串。
  • table.setn 函数已废弃。table.getn 对应新的长度运算符（#）；建议直接使用运算符而非函数。（详见 luaconf.h 的 LUA_COMPAT_GETN 选项。）
  • loadlib 函数重命名为 package.loadlib。（详见 luaconf.h 的 LUA_COMPAT_LOADLIB 选项。）
  • math.mod 函数重命名为 math.fmod。（详见 luaconf.h 的 LUA_COMPAT_MOD 选项。）
  • table.foreach 和 table.foreachi 函数已废弃。可用 pairs 或 ipairs 的 for 循环替代。
  • require 函数因新模块系统有较大变化。新行为与旧版大致兼容，但 require 现在从 package.path 获取路径，而非 LUA_PATH。
  • collectgarbage 函数参数有变化。gcinfo 函数已废弃；请用 collectgarbage("count") 替代。

7.3 - API 的变化

  • luaopen_* 系列函数（用于打开库）不能像普通 C 函数那样直接调用，必须通过 Lua 以 Lua 函数方式调用。
  • lua_open 函数被 lua_newstate 替代，允许用户自定义内存分配函数。可用标准库的 luaL_newstate 创建基于 realloc 的标准分配函数的状态。
  • 辅助库中的 luaL_getn 和 luaL_setn 函数已废弃。请用 lua_objlen 替代 luaL_getn，luaL_setn 无需替代。
  • luaL_openlib 函数被 luaL_register 替代。
  • luaL_checkudata 现在在给定值不是期望类型的 userdata 时会抛出错误（Lua 5.0 返回 NULL）。

8 - Lua 的完整语法

以下是 Lua 的完整语法，采用扩展 BNF 表示法。 （不包括运算符优先级的描述。） 


    chunk ::= {stat [`;´]} [laststat [`;´]]

    block ::= chunk

    stat ::=  varlist `=´ explist | 
             functioncall | 
             do block end | 
             while exp do block end | 
             repeat block until exp | 
             if exp then block {elseif exp then block} [else block] end | 
             for Name `=´ exp `,´ exp [`,´ exp] do block end | 
             for namelist in explist do block end | 
             function funcname funcbody | 
             local function Name funcbody | 
             local namelist [`=´ explist] 

    laststat ::= return [explist] | break

    funcname ::= Name {`.´ Name} [`:´ Name]

    varlist ::= var {`,´ var}

    var ::=  Name | prefixexp `[´ exp `]´ | prefixexp `.´ Name 

    namelist ::= Name {`,´ Name}

    explist ::= {exp `,´} exp

    exp ::=  nil | false | true | Number | String | `...´ | function | 
             prefixexp | tableconstructor | exp binop exp | unop exp 

    prefixexp ::= var | functioncall | `(´ exp `)´

    functioncall ::=  prefixexp args | prefixexp `:´ Name args 

    args ::=  `(´ [explist] `)´ | tableconstructor | String 

    function ::= function funcbody

    funcbody ::= `(´ [parlist] `)´ block end

    parlist ::= namelist [`,´ `...´] | `...´

    tableconstructor ::= `{´ [fieldlist] `}´

    fieldlist ::= field {fieldsep field} [fieldsep]

    field ::= `[´ exp `]´ `=´ exp | Name `=´ exp | exp

    fieldsep ::= `,´ | `;´

    binop ::= `+´ | `-´ | `*´ | `/´ | `^´ | `%´ | `..´ | 
             `<´ | `<=´ | `>´ | `>=´ | `==´ | `~=´ | 
             and | or

    unop ::= `-´ | not | `#´
